# 反义词对自动发现工具

自动发现反义词对，用于扩展定律验证的词表。

## 快速开始

```bash
# 运行所有方法
python opposites.py --method all --top-k 20

# 单独运行某种方法
python opposites.py --method projection --top-k 20
python opposites.py --method clustering --top-k 20
python opposites.py --method llm --top-k 20
```

## 命令行参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--model` | str | `infoxlm-large` | 语言模型 |
| `--layer` | str | `0` | 提取层 |
| `--method` | str | `all` | 发现方法 |
| `--top-k` | int | `20` | 返回结果数量 |

### 方法类型

| 值 | 说明 |
|----|------|
| `projection` | 语义轴投影法 |
| `clustering` | 向量聚类法 |
| `llm` | 模型判断法 |
| `all` | 运行所有方法并组合结果 |

## 三种方法原理

### 1. 语义轴投影法（推荐）

**原理**：从已知反义词对学习语义轴方向，然后在新词对上找投影相反的词。

**步骤**：
1. 从已知反义词对 (pos, neg) 计算轴方向：`axis = v_pos - v_neg`
2. 用 PCA 正交化多个轴方向，得到语义基
3. 将词汇投影到语义轴上
4. 找投影值符号相反且差距大的词对

**优势**：
- 利用已知知识指导发现
- 找到的是语义上真正相反的词
- Recall: 4/18 (22%)

**示例输出**：
```
奴役-自由: 6.491 ✓ KNOWN
存在-虚无: 5.101 ✓ KNOWN
正义-不公: 4.946 ✓ KNOWN
知识-无知: 4.937 ✓ KNOWN
```

### 2. 向量聚类法

**原理**：将词向量聚类，在跨簇词对中找语义相反的词。

**步骤**：
1. 对词向量做 K-means 聚类
2. 从已知反义词对学习评价轴
3. 对跨簇词对计算"相反程度"分数
4. 分数 = (1-相似度) × 0.4 + 投影分离 × 0.4 + 模长比 × 0.2

**优势**：
- 不依赖单一维度
- 综合多个指标判断

**局限**：
- 需要足够的已知反义词对来学习评价轴
- Recall: 0/18

### 3. 模型判断法

**原理**：结合语义轴投影和多标准判断。

**步骤**：
1. 从已知反义词对学习评价轴
2. 生成候选词对（投影值差距大的）
3. 多标准判断：
   - 余弦相似度 < -0.3
   - 投影分离 > 1.0
   - 满足任一条件即为反义词

**优势**：
- 自动生成候选，减少计算量
- 多标准提高准确性

**局限**：
- 依赖已知反义词对质量
- Recall: 1/18

## 输出说明

```
======================================================================
Projection-based Discovery Results
======================================================================
 1. 奴役-自由: 6.491 ✓ KNOWN
 2. 奴役-虚无: 5.859
 3. 奴役-无知: 5.517
 ...

Recall: 4/18 known pairs found
```

- **✓ KNOWN**：表示是已知的反义词对
- **Recall**：发现的已知反义词对数量 / 总已知数量

## 方法对比

| 方法 | Recall | 发现新词对 | 计算速度 | 适用场景 |
|------|--------|-----------|---------|---------|
| 投影法 | **4/18** | 多 | 快 | 有已知反义词对时 |
| 聚类法 | 0/18 | 少 | 中 | 词表大时 |
| 模型法 | 1/18 | 中 | 慢 | 需要精准判断时 |

## 扩展词表

发现新的反义词对后，可添加到定律配置中：

```yaml
# laws/law1.yaml
opposite_pairs:
  - [完备, 不完备]
  - [正确, 错误]
  # 新发现的词对
  - [正义, 虚无]
  - [奴役, 虚无]
```

然后重新运行验证：

```bash
python geometry4.py --task all
```

## 原理详解

### 为什么投影法效果最好？

反义词的本质是：在某个语义维度上方向相反。

例如"好-坏"在价值轴上：
- 好 → 正向投影
- 坏 → 负向投影

从已知反义词对学习的轴，正是这个语义维度。新词对如果在这个轴上投影相反，很可能是反义词。

### 为什么聚类法效果差？

聚类基于向量相似度，但反义词往往相似度高（共享语法上下文）。

例如：
- cosine(好, 坏) ≈ 0.9
- cosine(对, 错) ≈ 0.8

反义词在聚类中往往在同一簇，跨簇配对反而找不到真正的反义词。

### 如何改进？

1. **扩大已知词对**：更多已知反义词对 → 更准确的语义轴
2. **多语言结合**：跨语言反义词对验证
3. **人机协作**：自动发现 + 人工确认
